## Queue Format Reference

When decisions are added, they follow this format:

```markdown
### DEC-YYYY-NNNN
- **Status**: pending | approved | rejected | deferred
- **Urgency**: critical | high | normal | low
- **Category**: Support | Ops | Growth | Finance
- **Received**: ISO timestamp
- **SLA**: Time until decision needed

**Summary**: One-line description

**Context**:
- Relevant background
- Customer/system info
- Related events

**AI Analysis**:
What the AI determined with reasoning.
Confidence: X%

**AI Recommendation**: What AI suggests

**Options**:
- a: Option A description
- b: Option B description
- c: Option C description
- d: Handle manually

**Decision**: [PENDING]
**Decided At**: -
**Decided By**: -
**Notes**: -
```
